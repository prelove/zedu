## ADDED Requirements

### Requirement: 全量端到端测试执行
系统必须在编译后的完整二进制上，通过真实HTTP请求（而非直接调用
service层）复测PRD第二十四章的全部测试用例。

#### Scenario: 账务事务类测试通过
- **WHEN** 在真实环境中执行TC-01~TC-07
- **THEN** 全部通过，包括课后确认的事务原子性、并发保护等场景

#### Scenario: 配置化边界测试通过
- **WHEN** 在真实环境中执行TC-08~TC-11
- **THEN** 全部通过，包括本位币锁定、支付方式/出勤分类字典的
  实时生效

#### Scenario: 幂等性与状态机测试通过
- **WHEN** 在真实环境中执行TC-12~TC-14
- **THEN** 全部通过，包括课前提醒不重复发送、课次状态转换限制

#### Scenario: 数据安全测试通过
- **WHEN** 在真实环境中执行TC-15~TC-17
- **THEN** 全部通过，包括未登录无法访问付款凭证、备份包含uploads目录

#### Scenario: 跨平台部署测试通过
- **WHEN** 在真实环境中执行TC-18~TC-21
- **THEN** 全部通过，三平台构建产物均可独立运行

### Requirement: 真实数据迁移
系统必须能将运营者现有的Excel数据完整、准确地导入生产环境。

#### Scenario: 导入总数核对
- **WHEN** 完成正式导入
- **THEN** 导入后的学生总数、老师总数与原Excel记录数（扣除因数据
  问题被跳过的行数）完全一致

#### Scenario: 抽样字段核对
- **WHEN** 随机抽取5~10个学生的姓名、联系方式等关键字段
- **THEN** 与原Excel对应记录完全一致

### Requirement: 备份恢复演练
系统必须验证一次完整的"备份→模拟故障→恢复"流程真实可行。

#### Scenario: 恢复后数据一致
- **WHEN** 执行备份、人为破坏数据库、再执行恢复
- **THEN** 恢复后系统正常启动，数据与备份时刻完全一致，且故障
  现场（损坏状态）在恢复前已被额外快照保存

### Requirement: 稳定性观察
系统必须能在24小时连续运行中验证定时任务按预期触发且无异常退出。

#### Scenario: 定时任务触发验证
- **WHEN** 观察24小时内的日志和notification_log记录
- **THEN** 六类定时任务均有对应的实际触发记录，触发频率符合各自
  的Cron配置

#### Scenario: 进程无异常退出
- **WHEN** 检查24小时观察期内的服务运行状态
- **THEN** 进程持续运行未因未捕获异常而退出，日志中无崩溃堆栈记录
