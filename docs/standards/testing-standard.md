# 测试规范

- TDD：先运行并保存失败证据，再做最小实现，重构后全量复验。
- Unit覆盖纯规则、状态机、校验、权限、i18n、幂等；新增代码≥80%，财务/认证/RBAC/上传/通知关键分支100%。
- Integration使用真实SQLite临时库、migration、fake Resend和临时文件目录，覆盖并发、故障注入与补偿。
- E2E默认Playwright；Windows原生安装/文件选择器等用computer-use补充。
- 测试不得向真实用户发信；真实Resend只使用批准测试收件箱。
- 不允许删除测试、放宽断言、跳过失败场景来让门禁通过。

## 执行分层

- 每个小切片的本机必跑门禁是格式/静态检查、`go test ./... -count=1`（或前端对应的单次 lint/typecheck/unit/build）以及该切片的定向并发、故障注入和事务回归。不得用重复次数替代针对性的断言。
- `go test ./... -count=20` 是稳定性扫描，不是每个小任务的常规门禁；仅在里程碑候选验收、迁移/并发基础设施变更、或曾出现非确定性失败时执行，并将结果写入验收证据。
- 后端涉及共享状态、认证、事务或并发时，Linux CI 必须运行一次 `go test ./... -race -count=1`；Windows 本机因 CGO 限制不能伪造 race 结果。
