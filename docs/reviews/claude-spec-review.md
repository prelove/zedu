# Claude PRD/OpenSpec 评审报告

## 总结

PRD v3.1 的业务原则、数据深度和风险意识较好，可作为事实源主体；旧 001-014 具有较高素材价值，但不能直接执行。系统性问题包括：OpenSpec 1.6 格式不合规、change 横跨多个能力、MVP/V1混合、部分场景只验证字段存在、测试和上线过程被当成功能规格。

## 必须修正的重大问题

1. 旧通知失败仍设置 `remind_sent_at/REMINDED`，会制造假成功。
2. 自动将过期课次置 `COMPLETED` 与“完成即财务确认”冲突，应改为待确认/逾期语义。
3. 备份只含 SQLite 与 MVP 新增凭证冲突。
4. 仅登录即可访问凭证缺少对象授权，存在 IDOR 风险。
5. 正式结款与排课增强混在同一 change，破坏 MVP 边界。

## 结论

采用“冻结旧稿、按业务能力重写”的方案。任何旧 task 只有迁移到新 Requirement/Scenario/Test 并通过严格校验后才能进入 READY。
